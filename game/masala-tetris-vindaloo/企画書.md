## 1. DXからの進化ポイント（なぜ続編か）

DXの天井は「速くなるだけのクラシックテトリス」。スコアは消去ライン依存で戦略の深さと"もう一回"の理由が薄く、AIは実況のみでゲームに絡まなかった。VINDALOOは以下で殻を破る:

1. **役（コンボ）システム** ― ラインを消すとスパイスが鍋に溜まり、組み合わせで「カレーが完成」して大連鎖・大スコア。何を残し何を消すかの判断ゲームに。
2. **ローグライト進行** ― ステージ間に「スパイス商人」が登場。レリック/ミノ強化を購入し、毎回違うビルドで挑む中毒構造。
3. **AI対戦（VERSUS）** ― ローカル盤面評価AIと画面分割対戦。お邪魔ブロックの送り合い。DXのAIキャラ（マハラジャ/仏陀/シヴァ）が"敵"として活きる。
4. **競技レベルの演出＆操作性** ― SRS回転・ホールド・ゴースト・T-spin・REN・パーティクル・画面震え・グロー。Tetris Effect 風の没入。

## 2. ゲームモード

- **CURRY∞（メイン / 1人用ローグライト）**
  ステージ制。各ステージは時間制 or ノルマ制。クリア毎に商人フェーズ。スパイス役で連鎖を狙う。
- **VERSUS（対AI対戦）**
  画面分割。ローカルAIと同時進行でライン消去→お邪魔送り合い。AIの性格＝評価重みで強さと戦法が変わる。
- **隠しモード（DX踏襲）**
  特定プレイヤー名で発動する激辛/慈悲モード。DXの `SECRET_NAMES` / `EASY_NAMES` 文化を継承。

## 3. コアシステム詳細

### 3.1 スパイス役システム
7種のスパイスを標準テトロミノに紐づけ、ミノ自体がスパイス属性を持つ。

| ミノ | スパイス | 色 |
|---|---|---|
| I | チリ 🌶️ | 赤 |
| O | ターメリック | 黄 |
| T | ガラムマサラ | 紫 |
| L | クミン | 茶 |
| J | サフラン | 金 |
| S | カルダモン | 緑 |
| Z | コリアンダー | 薄緑 |

- ライン消去時、そのラインに含まれたスパイスが「鍋ゲージ」に蓄積。
- 鍋の蓄積が**役テーブル**の条件を満たすと「カレー完成」→ 大連鎖演出＋スコア倍率＋お邪魔送り（対戦時）→ 鍋リセット。

**役テーブル（v1：5種）**

| 役 | 条件（鍋の蓄積スパイス） | 効果 |
|---|---|---|
| マイルド | 異なる3種以上 | スコア ×1.5 |
| スパイシー | 同種3つ | ×2 ＋ お邪魔1段送り |
| ビリヤニ | 5種以上 | ×3 ＋ 鍋リセットボーナス |
| ヴィンダルー | チリ🌶️×3 | ×3 ＋ お邪魔3段（最高単独役） |
| ターリー | 全7種コンプ | ×5 ＋ 相手に大量お邪魔（全消し級） |

### 3.2 ローグライト（商人フェーズ）
- ステージのノルマ（指定役をN回完成）達成毎に、スコア（＝ルピー）で購入する商人フェーズへ。

**レリック初期ラインナップ（v1：6種）**

| レリック | 効果 |
|---|---|
| 激辛壺 | チリ消去時の連鎖 +2 |
| 第二の手 | ホールド枠を2に |
| 黄金スプーン | ソフトドロップ1段ごとに微加点 |
| 不死鳥の灰 | 1回だけ gameover 回避 |
| 商人の天秤 | 商人の品揃え +1・割引 |
| シヴァの三叉 | 必殺ゲージの蓄積が速くなる |

- v1はこの規模で組み、面白さを確認してから役・レリックを拡張する。
- 商人の品揃え口上を **LLM実況（4回のうち1回）** で生成。毎回違う味。

### 3.3 対戦AI（ローカル評価関数）
```
各ミノについて「全配置 × 全回転」を列挙
 → 仮置き盤面を4指標でスコアリング
    ・aggregate height（低いほど良い）
    ・holes（少ないほど良い）
    ・bumpiness（平らなほど良い）
    ・lines cleared（多いほど良い）
 → 最高スコア手を選択して着手
```
- 強さ＝重み係数＋思考の隙（弱いAIはわざと空洞を作る/反応を遅らせる）。
- 性格：マハラジャ＝バランス、シヴァ＝攻撃特化（お邪魔ガン送り）、仏陀＝守り型。**同一エンジンの重み変更だけでキャラが立つ。**

### 3.4 標準テトリス機能の競技化
- SRS（Super Rotation System）準拠の回転＋壁蹴り。
- ホールド機能、ゴーストピース、7-bag ランダマイザ、ハードドロップ、ソフトドロップ、REN（連続消去ボーナス）、T-spin判定。

## 4. LLM実況の設計（1プレイ4回前後 / Gemini継続）

頻発イベント（移動・回転・連鎖中）は **Web Audio効果音＋ローカル定型文** で即応。LLMは"ここぞ"の非同期4箇所のみ：

| # | タイミング | 内容 |
|---|---|---|
| 1 | ゲーム開始 | キャラ別の煽り |
| 2 | 商人フェーズ | 品揃えの口上 |
| 3 | 特大連鎖 / 役完成 | 絶賛コメント |
| 4 | ゲームオーバー / クリア | 総括コメント |

- 既存ワーカー `POST /api/gemini` をそのまま使用。fetch失敗時はローカル定型文にフォールバック（DXの `aiSay` 方式踏襲）。
- 待ち時間はゲーム進行をブロックしない（非同期）。
- DeepSeek等への切替が将来必要になっても、ワーカーにエンドポイントを1本足してfetch先を変えるだけで対応可能。

## 6. レイアウト方針（DX両対応を踏襲）

- **PC**：中央ゲーム盤＋左右サイドバー（スコア/タイム/NEXT/HOLD/役一覧/レリック）。VERSUSは2盤面横並び。
- **スマホ**：縦積み。上部ステータスバー、下部固定タッチコントローラ（十字＋DROP＋必殺）。VERSUSは自盤面メイン＋相手盤面ミニ表示。
- DXの `@media` ブレークポイント（768px）とコントローラUIを再利用。

## 7. 技術構成

- 単一 `index.html`（Tailwind CDN, Google Fonts, Font Awesome）。
- Canvas 2D 描画（DXの 20px/block スケール方式を踏襲・拡張）。
- 状態管理は素のJS。ローグライト/役/AIをモジュール的に関数分割。
- ランキングは `localStorage`（DX踏襲）。SNSシェア（X/LINE）も踏襲。
- SEO/OGP/JSON-LD は DX 同形式で `masala-tetris-vindaloo` 用に整備。
